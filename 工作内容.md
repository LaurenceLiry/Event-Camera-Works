# 工作内容

## Stage 0（1–2天）：明确工程输入输出

目标：

- 明确系统输入 / 输出
- 选一个“最简单可控”的场景
    - 室内
    - 一根固定不动的细绳/钢丝
    - 背景简单
    - 手持事件相机或装在无人机上，绕线运动
- 决定第一版“不用深度学习”，用纯几何方法

## Stage 1（1–2周）：先把“2D线事件”提取出来

**目标**：从事件流中稳定提取“属于细线的事件点/像素”

- 先不要重写Skyshield / Lattice，第一步直接用成熟思路：
    - STC（时空对比）滤波 → 降噪
    - time surface / SAE（Surface of Active Events）
    - 在短时间窗口内累积事件

- 选一个“最容易实现”的2D线检测方式，有三种选择，推荐从最简单的开始：
    - 方案A（最快）：Hough / RANSAC
    - 方案B：时空平面拟合（Powerline思路）
    - 方案C：轻量网络 / Lattice

**Stage 1验收标准**：证明在连续时间里，同一根细线能被反复检测到（即使有噪声）。

## Stage 2（核心创新，2–3周）：用位姿做3D重建

**目标**：2D线映射到3D

- 把2D线变成3D射线

- 多视角几何：射线聚合，现在有很多射线，它们应该在空间中靠近同一条3D线
    - 用最小二乘 / PCA / RANSAC
    - 拟合一条3D直线或分段直线
    - 或先得到3D点云 → 再拟合曲线

- 做一个最简单的可视化
